# Corp Messenger — Что сделано и что осталось

## Краткая сводка
- **Текущий этап:** Этап 10/10 завершён (все этапы)
- **Фронтенд:** React 19 + Vite 6 + React Router 6
- **Бэкенд:** Express + Socket.IO + MySQL 8.4 (Laragon)
- **Демо:** http://localhost/messenger-app/

---

## ✅ Что сделано

### Этап 1 — База
- Авторизация (мок-логин с выбором пользователя)
- Сайдбар с навигацией (12 пунктов)
- Светлая тема, адаптивная вёрстка (десктоп + планшет + мобилка)
- 16 экранов: Чаты, Задачи, Сотрудники, Уведомления, Профиль, Проекты, Канбан, Гант, Сроки, Мой план, Файлы, Админ-панель (5 вкладок)
- Бэкенд: Express + Socket.IO на порту 3001
- MySQL 8.4 (Laragon, порт 3306, база corp_messenger)
- 10 таблиц: departments, users, chats, chat_members, messages, tasks, subtasks, notifications, push_subscriptions, campaigns
- Сид-данные (6 сотрудников, 4 отдела, 3 чата, 5 задач, 3 проекта)
- API: auth, chats, tasks, corporate, push, admin

### Этап 2 — Задачи и проекты
- Канбан-доска (3 колонки: todo/progress/done)
- Сроки (группировка по периодам)
- Мой план (задачи текущего пользователя)
- История изменений задачи
- Проекты (список + детальная с вкладками: задачи/чат/файлы/wiki)
- Диаграмма Ганта с масштабом день/неделя/месяц
- Чат внутри задачи
- Поиск сотрудников по отделу и должности

### Этап 3 — Личные сообщения
- Личные чаты (1-1) в общем списке
- Создание чата из справочника (кнопка «Написать»)
- Онлайн-статус (🟢 зелёная точка + «был(а) в …»)
- Индикатор печатания («печатает…»)
- Прочитано / не прочитано (✓✓)
- Редактирование сообщения (✏️ + пометка «ред.»)
- Удаление сообщения (🗑️)
- Реакции (7 emoji: 👍❤️😄😢😡🔥👀)
- Пересылка / закрепление / поиск по истории — UX готов

### Этап 4 — Администрирование
- Админ-панель `/admin` (4 вкладки: Дашборд, Пользователи, Отделы, Рассылки, Журнал)
- Управление пользователями (блокировка/разблокировка)
- Управление отделами (просмотр, список)
- Журнал действий (кто что делал)
- Роли: admin / manager / user (выбор при логине)
- Форма регистрации (имя, email, пароль, отдел)
- Сброс пароля (email с токеном — через Nodemailer)

### Этап 5 — Файлы
- Файловый диск `/files` (папки, поиск, сетка/список)
- Иконки по типу: 📄 PDF, 📝 DOCX, 📊 XLSX, 📽️ PPTX, 🖼️ PNG/JPG, 📦 ZIP
- Вложения в чатах (кнопка 📎, предпросмотр, lightbox)
- Вкладка «Файлы» в задачах
- Просмотр изображений (lightbox)

### Этап 6 — Уведомления
- Web Push (VAPID) — subscribe/unsubscribe, sendPushToUser, sendPushToAll
- Nodemailer — email-уведомления (сброс пароля, уведомления)
- Массовые рассылки (кампании) из админки
- Кнопка включения/отключения Push на странице уведомлений

### Этап 7 — Docker / Инфраструктура
- Dockerfile (multi-stage: Vite build → Node server)
- docker-compose.yml (MySQL 8 + сервер, инициализация схемы)
- `.env.example` — все переменные окружения
- `database/schema.sql` — полная схема для Docker
- `/api/health` — uptime, rid, статус
- JSON-логирование (logger.js из SwiftMatch)

### Этап 8 — Безопасность
- **AES-GCM шифрование** сообщений (`src/lib/crypto.js` из SwiftMatch)
- **Клиентский фильтр** мата/спама (`src/lib/word-filter.js` из SwiftMatch)
- **Серверный кеш** запрещённых слов (`server/src/banned-words.js` из SwiftMatch)
- Helmet + rate-limit + CORS
- JWT auth + optionalAuth middleware

### Этап 9 — Мобильное приложение
- Capacitor v8 — конфиг + Android-проект (собран `npx cap add android`)
- PWA — `manifest.json` + Service Worker (`sw.js`) + кеширование + Push-уведомления
- `PwaRegistry.jsx` — регистрация SW
- `PwaBanner.jsx` — баннер установки приложения
- `native.js` — Capacitor-хелперы (камера, API base)

### Этап 10 — AI и коллаборация
- `ai.js` — 15 айсбрейкеров (темы для разговора) + AI-подсказки подзадач по ключевым словам
- AI-генерация плана задачи по названию (отчёт, встреча, письмо и т.д.)

### Архитектурные улучшения (из SwiftMatch)
- `server/src/ws.js` — Socket.IO с JWT-авторизацией и комнатами `user:{id}`
- `src/hooks/useWebSocket.js` — хук с авто-реконнектом (10 попыток)
- `src/hooks/useAntiScreenshot.js` — защита от скриншотов (блокировка контекстного меню, PrintScreen, Ctrl+C)
- `src/lib/api.js` — ApiClient с авто-refresh токенов (JWT refresh), retry с exponential backoff
- Внешний вид: полный редизайн (фиолетовый градиент, анимации, тени)

## ✅ Дополнительно (доделано)
- **JWT refresh tokens** — сервер выдаёт access (24h) + refresh (30d), клиент авто-обновляет при 401
- **Anti-screenshot** — хук `useAntiScreenshot.js` (блокировка контекстного меню, PrintScreen, F12, Ctrl+C/S/P)
- **Онлайн-доска** (`/board`) — HTML5 Canvas с кистью, ластиком, 8 цветов, 5 размеров, undo, экспорт PNG, touch
- **Совместное редактирование досок** — Socket.IO `board:draw` / `board:clear`, хук `useBoardSync.js`
- **Интеллект-карты** (`/mindmap`) — canvas с узлами, связями (стрелки), drag-and-drop, редактирование текста, zoom, экспорт SVG, двойной клик для добавления узла, Delete для удаления
- **2FA (TOTP)** — speakeasy + qrcode на сервере, страница `/2fa` для настройки/отключения, QR-код для Google Authenticator, проверка при логине
- **Регистрация** — страница Register.jsx + API `/api/auth/register`, bcrypt, авто-логин
